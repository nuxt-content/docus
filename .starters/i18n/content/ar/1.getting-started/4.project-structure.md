## البنية العامة

Docus هو **طبقة Nuxt** تمدد تطبيق Nuxt القياسي بمزايا خاصة بالتوثيق. هذا يمنحك مرونة مشروع Nuxt العادي مع نظام توثيق جاهز.

عند إنشاء مشروع Docus جديد باستخدام `npx create-docus my-docs`، ستكون البنية الأساسية كالتالي:

```bash
my-docs/
├── content/             # محتوى Markdown الخاص بك
│   ├── index.md         # الصفحة الرئيسية
│   └── docs/            # صفحات التوثيق
├── public/              # الموارد الثابتة
└── package.json         # الاعتماديات والسكربتات
```

يمكنك دائمًا استخدام أي ميزة أو ملف من مشروع Nuxt قياسي:

```bash
my-docs/
├── nuxt.config.ts       # إعدادات Nuxt (إضافة وحدات، مكونات...)
├── app/                 # مجلد التطبيق
    ├── app.config.ts    # إعدادات التطبيق
│   ├── components/      # المكونات (أضف مكوناتك الخاصة)
│   ├── layouts/         # التخطيطات (أضف تخطيطاتك الخاصة)
│   └── pages/           # الصفحات (أضف صفحاتك الخاصة)
└── server/              # كود جانب الخادم (أضف كود الخادم الخاص بك)
```

### مجلد `content/`

هذا هو المكان الذي تكتب فيه صفحاتك بصيغة Markdown. يقوم Docus بإنشاء المسارات تلقائيًا اعتمادًا على بنية ملفاتك.

**بنية أحادية اللغة:**

```bash
content/
├── index.md              # الصفحة الرئيسية (/)
├── getting-started.md    # صفحة توثيق (/getting-started)
└── guide/
    ├── introduction.md   # صفحة توثيق (/guide/introduction)
    └── configuration.md  # صفحة توثيق (/guide/configuration)
```

::tip
يمكنك فصل ملفات التوثيق داخل مجلد فرعي باسم `docs/` لجعلها متاحة عبر المسار `/docs`. كما يمكنك استبدال الصفحة الرئيسية باستخدام صفحات Vue مخصصة عند الحاجة. للمزيد راجع [توثيق التحرير](https://docus.dev/concepts/edition).
::

**بنية متعددة اللغات (مع i18n):**

```bash
content/
├── en/
│   ├── index.md             # الصفحة الرئيسية الإنجليزية (/en)
│   └── guide/
│       └── introduction.md  # صفحة توثيق (/en/guide/introduction)
└── ar/
    ├── index.md             # الصفحة الرئيسية العربية (/ar)
    └── guide/
        └── introduction.md  # صفحة توثيق (/ar/guide/introduction)
```

::tip{to="https://docus.dev/concepts/internationalization"}
تتوفر معلومات إضافية عن i18n في قسم التدويل.
::

### مجلد `public/`

الملفات الموجودة داخل مجلد `public/` تُقدَّم من جذر الموقع ولا يتم تعديلها أثناء عملية البناء. هذا هو المكان المناسب لوضع الصور والأيقونات وباقي الموارد الثابتة.

### `package.json`

يحتوي هذا الملف على كل اعتماديات التطبيق وسكربتاته. ملف `package.json` في تطبيق Docus يكون بسيطًا جدًا ويشبه التالي:

```json [package.json]
{
  "name": "my-docs",
  "scripts": {
    "build": "nuxt build --extends docus",
    "dev": "nuxt dev --extends docus",
  },
  "dependencies": {
    "docus": "latest",
    "better-sqlite3": "^12.2.0",
    "nuxt": "^4.0.0"
  }
}
```

### `nuxt.config.ts`

*هذا الملف ليس إلزاميًا لبدء تطبيق Docus.*

يمكنك إضافة وحدات إضافية إلى ملف إعدادات Nuxt:

```typescript [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['@vercel/analytics/nuxt/module']
})
```

### `app.config.ts`

*هذا الملف ليس إلزاميًا لبدء تطبيق Docus.*

::warning
يجب أن يكون ملف `nuxt.config.ts` موجودًا حتى تتمكن من تجاوز إعدادات التطبيق. بدون ملف إعدادات Nuxt لن يتم أخذ التجاوزات الخاصة بك في الاعتبار.
::

من هنا يمكنك إعداد Docus ليتوافق مع علامتك، وإدارة SEO، وتحديد اللغة، وضبط الروابط وحسابات التواصل الاجتماعي. راجع [توثيق الإعدادات](https://docus.dev/concepts/configuration) لمزيد من التفاصيل.

```ts [app.config.ts]
export default defineAppConfig({
  docus: {
    locale: 'ar', // تحديد اللغة عند استخدام لغة واحدة
  },
  seo: {
    title: 'توثيقي',
    description: 'توثيق رائع خاص بي',
  },
  // ... إعدادات أخرى
})
```

## البنية الكاملة لمشروع Nuxt

بما أن Docus هو طبقة Nuxt، يمكنك استخدام **أي ميزة** من ميزات مشروع Nuxt القياسي:

::warning
يجب أن يكون ملف `nuxt.config.ts` موجودًا حتى تتمكن من تجاوز إعدادات تطبيق Nuxt. بدون ملف إعدادات Nuxt لن يتم أخذ التجاوزات في الاعتبار.
::

```bash
my-docs/
├── app/                 # مجلد التطبيق (اختياري)
│   ├── app.config.ts    # إعدادات التطبيق
│   ├── components/      # مكونات Vue مخصصة
│   ├── layouts/         # تخطيطات مخصصة
│   ├── pages/           # صفحات Vue مخصصة (خارج المحتوى)
│   ├── composables/     # Composables خاصة بـ Vue
│   └── middleware/      # Middleware للمسارات
├── server/              # كود جانب الخادم
│   └── api/             # مسارات API
├── plugins/             # إضافات Nuxt
├── middleware/          # Middleware عام
└── modules/             # وحدات Nuxt مخصصة
```

::tip{to="https://docus.dev/concepts/nuxt"}
هذا النهج القائم على الطبقات يمنحك كل مزايا مشروع Nuxt القياسي.
::
