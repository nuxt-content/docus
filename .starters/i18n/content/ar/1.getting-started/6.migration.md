## **الترحيل من Docus v3 إلى v4**

يقدم Docus v4 إصدارًا جديدًا يعتمد على واجهة أوامر Nuxt بدلًا من واجهة أوامر Docus. سيظل المحتوى والإعدادات الحالية متوافقة، وكل ما تحتاجه هو تحديث الأمر المستخدم لتشغيل الخادم أو عملية البناء.

### ⚠️ تغييرات كاسرة

أهم التغييرات الكاسرة مرتبطة بأوامر CLI:

| v3                       | v4                          |
| ------------------------ | --------------------------- |
| `npx docus init my-docs` | `npx create-docus my-docs`  |
| `docus dev`              | `nuxt dev --extends docus`   |
| `docus build`            | `nuxt build --extends docus` |

::tip
سيعمل محتوى Markdown الحالي بدون تغييرات. الترحيل يتعلق أساسًا بتحديث سير عمل التطوير والبناء.
::

## **الترحيل إلى Docus**

هل تستخدم بالفعل حلًا قائمًا على Markdown لتوثيقك؟ سواء كان **Docus v1** أو **قالب توثيق Nuxt UI** أو أي حل آخر لموقع ثابت، فالانتقال إلى Docus بسيط ومباشر.

يوفر Docus حلًا نظيفًا وسهل الصيانة باعتمادية واحدة فقط: مكتبة Docus نفسها. لن تحتاج إلى إدارة اعتماديات متعددة. كل شيء مدمج ويتم صيانته معًا، مما يجعل تحديث التوثيق أسهل.

للترحيل، انقل ملفات Markdown الحالية ببساطة إلى مجلد `content/` داخل مشروع Docus الأساسي.

بعد ذلك سيكون لديك سيناريوهان:

- **إذا كان توثيقك الحالي يستخدم بالفعل Nuxt Content وصيغة MDC**، فتأكد أن المكونات المستخدمة موجودة في Nuxt UI. وإذا كانت بعض المكونات غير موجودة، يمكنك إنشاء مكوناتك الخاصة بسهولة.
- **إذا كنت تستخدم Markdown قياسيًا**، يمكنك نسخ ملفاتك كما هي. بعد ذلك حسّن التوثيق تدريجيًا باستخدام المكونات المدمجة التي يوفرها Nuxt UI.

بعد نقل محتواك إلى مجلد `content/`، يمكنك تخصيص تطبيقك باتباع [توثيق الإعدادات](https://docus.dev/concepts/configuration).

صُمم Docus للتركيز على كتابة المحتوى، لذلك إذا كنت تستخدم Markdown بالفعل فيمكنك الانتقال إلى Docus بسهولة.
